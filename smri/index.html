üìã SMRI Notation Standard

Format: S{M}.{RRR}.{II}

Components:
S   = Scenario prefix (always "S")
M   = Module owner (1-6 internal, 11+ external)
RRR = Relations (comma-separated for multiple)
II  = Instance number (01-99)

Module Numbers:
1 = Shop          | 11 = Cloudflare (11.1=KV, 11.2=Workers, 11.3=CDN)
2 = Game          | 12 = Stripe (12.1=Checkout, 12.2=Webhooks, 12.3=API)
3 = Auth          | 13 = GitHub (13.1=Pages, 13.2=Actions)
4 = Payment       |
5 = Worker        |
6 = Common        |

Relation Notation:
0              ‚Üí Standalone (no relations)
2              ‚Üí Single relation (to Game)
2,3,4          ‚Üí Multiple internal (Game, Auth, Payment)
11.2-12.2      ‚Üí External sub-services (CF.KV + Stripe.Webhooks)
2-11.2-12.2    ‚Üí Mixed (Game + CF.KV + Stripe.Webhooks)
